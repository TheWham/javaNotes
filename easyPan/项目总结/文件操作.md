## 文件的分片上传操作

> 具体思路: 上传的文件经过前端的切片, 将文件切成若干份大小相同的小片, 记录切片数. 并且将文件进行md5解析, 记录文件的MD5值.
>
> 后端需要接收的参数
>
> `fileId` 	string 文件ID 	  可选
>
> `file` 	  string 文件流 	  必需
>
> `fileName`   string 文件名 	  必需
>
> `filePid`    string 文件父id     必需  
>
> `fileMD5`    string 文件MD5值    必需  # 用于判断是否是同一个文件, 若是同一个文件则不用重复的存到服务器, 而是将文件的信息添加到数据库中即可, 即实现秒传.
>
> `chunkIndex` string 当前分片索引  必需
>
> `chunks`     string 总分片数     必需

```java
@Transactional(rollbackFor = Exception.class)
public UploadResultDTO uploadFile(SessionWebUserDTO webUserDTO, String fileId, MultipartFile file, String filePid, String fileName, String fileMD5, Integer chunkIndex, Integer chunks) {
    UploadResultDTO uploadResultDTO = new UploadResultDTO();
    boolean uploadSuccess = true;
    File newFile = null;
    try {
        if (StringTools.isEmpty(fileId)) {
            fileId = StringTools.generateRandomStr(Constants.USER_ID_LENGTH);
        }
        uploadResultDTO.setFileId(fileId);
        // userSpaceDTO存储云盘用户初始空间和可使用的总空间 通过userID获取
        UserSpaceDTO userSpaceDTO = redisComponent.getUserSpaceDTO(webUserDTO.getUserId()); 
        Date curDate = new Date();
        if (chunkIndex == 0) {
    		// 如果是第一片 根据fileMD5判断数据库中是否存在这一文件 如果存在则无需上传文件到服务器 直接更新相应的数据库数据即可
            FileInfoQuery query = new FileInfoQuery();
            query.setStatus(FileStatusEnums.USING.getStatus());
            query.setFileMd5(fileMD5);
            query.setSimplePage(new SimplePage(0, 1));
            List<FileInfo> fileList = fileInfoMapper.selectList(query);
            // 秒传
            if (!fileList.isEmpty()) {
                FileInfo fileInfo = fileList.get(0);
                if (fileInfo.getFileSize() + userSpaceDTO.getUseSpace() > userSpaceDTO.getTotalSpace()) {
                    throw new BusinessException(ResponseCodeEnum.CODE_904);
                }
                fileInfo.setFileId(fileId);
                fileInfo.setFilePid(filePid);
                fileInfo.setUserId(webUserDTO.getUserId());
                fileInfo.setCreateTime(curDate);
                fileInfo.setLastUpdateTime(curDate);
                fileInfo.setDelFlag(FileDelFlagEnums.USING.getFlag());
                fileInfo.setFileMd5(fileMD5);
                fileName = autoUpdateFileName(webUserDTO.getUserId(), filePid, fileName);
                fileInfo.setFileName(fileName);
                this.fileInfoMapper.insert(fileInfo);
                uploadResultDTO.setStatus(UploadStatusEnums.UPLOAD_SECONDS.getCode());
                //更新用户使用空间
                updateUserUseSpace(webUserDTO.getUserId(), fileInfo.getFileSize());
                return uploadResultDTO;
            }
        }
         // 为了计算上传分片的总和 判断是否上传完毕或者需要继续上传 
        // 将已上传分片大小总和存入redis中 方便计算下一分片
        // 获取目前上传文件总和
        Long tempSize = redisComponent.getFileTempSize(webUserDTO.getUserId(), fileId);
        // 判断是否已将文件上传完毕
        if (tempSize + file.getSize() + userSpaceDTO.getUseSpace() > userSpaceDTO.getTotalSpace()) {
            throw new BusinessException(ResponseCodeEnum.CODE_904);
        }
        
        String fileTempFolder = appConfig.getBaseFileFolder() + Constants.FILE_FOLDER_NAME + Constants.FILE_FOLDER_TEMP;
        String currentUserFileFolder = webUserDTO.getUserId() + fileId;
        File uploadFile = new File(fileTempFolder + currentUserFileFolder);
        if (!uploadFile.exists()) {
            uploadFile.mkdirs();
        }
        newFile = new File(uploadFile.getPath() + "/" + chunkIndex);
        file.transferTo(newFile);
        redisComponent.saveFileTempSize(webUserDTO.getUserId(), fileId, file.getSize());
        if (chunkIndex < chunks - 1) {
            // 如果上传完毕 更新上传状态 并返回上传信息
            uploadResultDTO.setStatus(UploadStatusEnums.UPLOADING.getCode());
            return uploadResultDTO;
        }
        
        // 文件上传完毕之后, 加入数据库, 并进行数据异步合并
        String mouthFolder = DateUtils.format(curDate, DateTimePatternEnum.YYYYMM.getPattern());
        String realFileName = currentUserFileFolder + StringTools.getFileNameSuffix(fileName);
        FileTypeEnums fileTypeEnums = FileTypeEnums.getFileTypeBySuffix(StringTools.getFileNameSuffix(fileName));
        fileName = autoUpdateFileName(webUserDTO.getUserId(), filePid, fileName);
        FileInfo fileInfo = new FileInfo();
        fileInfo.setFileName(fileName);
        fileInfo.setFilePid(filePid);
        fileInfo.setFileMd5(fileMD5);
        fileInfo.setFilePath(mouthFolder + "/" + realFileName);
        fileInfo.setLastUpdateTime(curDate);
        fileInfo.setCreateTime(curDate);
        fileInfo.setFolderType(FileFolderTypeEnums.FILE.getType());
        fileInfo.setFileType(fileTypeEnums.getType());
        fileInfo.setFileSize(redisComponent.getFileTempSize(webUserDTO.getUserId(), fileId));
        fileInfo.setDelFlag(FileDelFlagEnums.USING.getFlag());
        fileInfo.setStatus(FileStatusEnums.TRANSFER.getStatus());
        fileInfo.setFileCategory(fileTypeEnums.getCategory().getCategory());
        fileInfo.setUserId(webUserDTO.getUserId());
        fileInfo.setFileId(fileId);
        fileInfoMapper.insert(fileInfo);
        Long fileTempSize = redisComponent.getFileTempSize(webUserDTO.getUserId(), fileId);
        //更新用户已使用空间
        updateUserUseSpace(webUserDTO.getUserId(), fileTempSize);
        uploadResultDTO.setStatus(UploadStatusEnums.UPLOAD_FINISH.getCode());
        // 事务提交之后执行
        TransactionSynchronizationManager.registerSynchronization(new TransactionSynchronization() {
            @Override
            public void afterCommit() { 
                // 将上传的文件存储到服务器上面 并生成相应视频的缩略图 以及视频的切割合并操作
                fileInfoService.transfer(fileInfo.getFileId(), webUserDTO);
            }
        });
    } catch (BusinessException e) {
        log.error("文件上传失败");
        uploadSuccess = false;
        throw e;
    } catch (Exception e) {
        log.error("文件上传失败");
        uploadSuccess = false;
    } finally {
        if (!uploadSuccess && newFile != null) {
            try {
                FileUtils.deleteDirectory(newFile);
            } catch (IOException e) {
                log.error("删除临时文件目录失败", e);
            }
        }
    }
    return uploadResultDTO;
}
```

### @Async注解作用

> ​	`@Async` 是 Spring 提供的注解，用于 **异步执行方法**，即将方法的执行放入一个独立的线程中进行，从而实现 **非阻塞操作**。
>
> 当一个方法被 `@Async` 标注时：
>
> 1. Spring 会将方法的执行委托给一个 **线程池** 中的线程执行，而不是使用调用线程。
> 2. 方法会在后台异步运行，调用者不会等待其执行完成。
>
> ------
>
> ### 详细说明
>
> 1. **异步方法执行**
>    - 被标注了 `@Async` 的方法会立即返回，执行线程由线程池中的一个线程来处理任务。
>    - 适合耗时较长但与主线程逻辑解耦的任务。
> 2. **线程池的使用**
>    - 默认情况下，Spring 会使用一个 **SimpleAsyncTaskExecutor** 来运行异步任务。
>    - 可以自定义线程池，例如使用 `ThreadPoolTaskExecutor`，提高任务执行效率。
> 3. **非阻塞行为**
>    - 主线程执行 `@Async` 方法时不会等待方法执行完成，而是继续执行主线程中的逻辑。
> 4. **增强 Spring 管理对象**
>    - 异步执行方法必须通过 Spring **代理对象** 调用，而不是直接调用类中的方法，否则 `@Async` 将失效。
